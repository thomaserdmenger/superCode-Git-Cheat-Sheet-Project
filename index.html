Git Cheat Sheet

Was Git ist

  • System, das Änderungen an unseren Dateien aufzeichnet und sichert
  • Zugriff auf alle Datei-Versionen möglich
  • Zurücksetzen der Dateien auf frühere Versionen möglich
  • Arbeitsteilige Zusammenarbeit von Entwicklern an einem Projekt möglich
  • Entwickler haben eigene Version der Projektdateien auf ihrem Computer

Basic Commands

git init: Erstellen eines leeren Git-Repositorys / Reinitialisieren eines vorhanden Repopository

git clone http://..: Clonen eines Remote-Repository in ein neues Verzeichnis

git add .: Hinzufügen bereitgestellter Dateien zur Staging Area

git commit -m “message”: Übertragung der Inhalte der Staging Area als neuen Commit-Snapshot in das Repository

git push: Übertrage eine Branch zu Github

git log: Zeigt alle Commits der aktuellen Branch (ausführlich)

git log --oneline: Zeigt alle Commits der aktuellen Branch (komprimiert)

git status: Anzeigen geränderter Dateien im Arbeitsverzeichnis, die für die nächste Übertragung bereitgestellt werden

Branches

git branch: Vorhandene Branches anschauen

git branch branchname: Neue Branch erstellen

git checkout branchname: Wechsel zu einer anderen Branch

git checkout -b branchname: Branch erstellen und direkt dort hinwechseln

git branch -d branchname: Branch löschen (ausgehend von einer anderen Branch)

Mergen von Branches

  1. Wechsel zur Main Branch
  2. git merge branchname: Mergen von 2 Branches ausgehend von der Main Branch
  3. Pushen nach dem Mergen
  4. git branch -d branchname : Nicht mehr benötigte Branch löschen
  5. Pushen nach dem Löschen

Code auf spezifischen Commit zurücksetzen

  1. git log --oneline: Identifizieren des SHA
  2. git checkout commit_SHA .: Zurücksetzen auf spezifischen SHA
  3. Pushen in Repo nach dem Zurücksetzen
